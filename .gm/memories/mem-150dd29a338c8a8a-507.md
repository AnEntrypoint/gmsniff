exec_js spool bodies are JSON: a JS regex literal like /\r?\n/ inlined in the code string gets its escapes consumed at JSON-parse time (\r becomes a real CR), producing 'Unterminated regexp literal' in the spawned runtime. Reliable shapes: (a) write regex-free probe code (string includes/startsWith), or (b) Write the script to a scratch file and have exec_js run node <absolute path> -- never inline regex-bearing JS in a JSON dispatch body. Same class applies to any verb whose body is JSON-wrapped code.
